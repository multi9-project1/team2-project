# 🛠️ Map-si TI 데모 버전 오류 해결 및 최적화 기록

이 문서는 프로젝트 초기 상태에서 발생했던 주요 오류들을 해결하고, 정상적인 실시간 크롤링 및 AI 분석 결과를 출력하기 위해 수행된 **핵심 수정 사항**만을 정리한 기록입니다.

---

## 1. 프론트엔드 (Frontend) - 통신 및 데이터 매핑
*   **API 주소 고정 (`js/api.js`):**
    *   상대 경로 요청 시 5500번 포트로 잘못 가는 문제를 해결하기 위해 `utils.getApiBaseUrl()`을 사용하여 모든 요청을 `http://localhost:8000`으로 강제 고정.
*   **데이터 변환 로직 (`js/api.js`):**
    *   프론트엔드의 인덱스 기반 답변(0, 1, 2...)을 백엔드 모델인 `Qstyle_1`, `Qstyle_2` 형식으로 변환하고, 모든 값을 문자열(String)로 캐스팅하여 전송.
*   **API 함수 노출:**
    *   `fetchProfileFromState`, `request` 등 핵심 함수들이 `window.MapsiApi` 객체에 누락되지 않도록 통합 관리.

## 2. 백엔드 (Backend) - 유연성 및 안정성
*   **CORS 전면 개방 (`app/main.py`):**
    *   포트 불일치로 인한 차단을 막기 위해 `allow_origins=["*"]` 및 모든 메소드 허용.
*   **모델 유연화 (`app/main.py`):**
    *   `SurveyInputModel`에 `ConfigDict(extra='allow')`를 추가하여 정의되지 않은 필드가 들어와도 에러(422) 없이 무시하고 처리되도록 개선.
*   **파비콘 에러 방지 (`app/main.py`):**
    *   브라우저의 자동 요청인 `/favicon.ico`에 대해 204 No Content 응답 경로를 추가하여 불필요한 404 로그 차단.

## 3. 크롤러 (Crawlers) - 역추적 수집 기법
*   **범용 추출 로직 (`musinsa_crl.py`, `zigzag_crl.py`):**
    *   웹사이트의 클래스명(Selector)이 바뀌어도 작동하도록 **[이미지 찾기 -> 부모 노드 역추적 -> 텍스트 싹쓸이]** 기법 적용.
*   **지그재그 정밀 정제:**
    *   "앱으로 이동", "매일직진", "인기순" 등 상품이 아닌 시스템 문구 및 광고 배너를 강력한 블랙리스트와 정규식으로 필터링.
    *   `facebook.com/pixel` 등 광고 트래킹 이미지를 제외하고 `product-image` 도메인만 수집하도록 이미지 필터 강화.
*   **로딩 시간 최적화:**
    *   지그재그의 무거운 JS 로딩을 기다리기 위해 초기 대기 시간을 12초로 연장하고, 지연 로딩(Lazy Loading) 대응을 위해 단계별 스크롤 로직 추가.

## 4. 데이터셋 이미지 복구 (Static Files)
*   **경로 자동 추적 (`app/main.py` & `item_feature_builder.py`):**
    *   CSV에 이미지 경로가 없더라도 `item_id`가 파일명 형식이면 `sample_data` 폴더 전체를 검색(rglob)하여 실제 파일 위치를 찾아 정적 URL(`/sample-files/...`)을 생성하도록 보정.

## 5. 실시간 AI 유사도 평가 시스템 (CLIP API 연동)
*   **AI 마이크로서비스 연동 (`app/services/ai_similarity_service.py`):**
    *   외부 AI 서버(`ailon.iptime.org:53000`)와의 통신 모듈 구축.
    *   이미지 URL을 실시간으로 다운로드하여 Base64로 인코딩 후 AI 서버로 전송하는 파이프라인 구현.
    *   한국어 스타일 명칭을 AI 서버용 영어 라벨(예: `casual fashion`)로 자동 매핑.
*   **병렬 분석 최적화 (Speed Optimization):**
    *   `ThreadPoolExecutor`를 사용하여 최대 20개의 상품 이미지를 동시에 분석.
    *   순차 처리 대비 분석 속도를 약 5배 이상 단축(20개 기준 3~5초 내외 완료).
*   **순수 데이터 기반 정렬:**
    *   AI가 산출한 순수 Softmax 점수를 기반으로 상품을 재정렬하여, 단순 검색 순위가 아닌 사용자의 실제 스타일 DNA와 가장 일치하는 상품을 최상단에 배치.

## 6. 지그재그 크롤러 최종 고도화 (전수 조사 방식)
*   **이미지 기반 전수 조사 기법:**
    *   특정 클래스명이나 레이아웃에 의존하지 않고, 페이지 내 모든 이미지를 전수 조사하여 상품 후보군 추출.
    *   이미지 주소 필터링 강화: 트래킹 픽셀(`facebook`, `pixel`) 및 앱 로고(`static`, `icon`)를 도메인 단위로 원천 차단.
*   **상품 정보 무결성 검증:**
    *   사진(`<img>`) 발견 시 부모 노드를 최대 10단계까지 역추적하여 이름과 가격이 모두 포함된 '진짜 상품 상자'만 수집.
    *   텍스트 정제 로직 개선: "매일직진", "인기순", "₩기호 중복" 등 시스템 문구를 정규식으로 제거하여 순수 상품 정보만 추출.
*   **로딩 안정성 확보:**
    *   지그재그의 무거운 동적 스크립트 로딩을 위해 초기 대기 시간을 12초로 연장하고 일정한 간격의 스크롤 적용.

## 7. 시스템 통합 및 에러 핸들링
*   **데이터 파이프라인 완성:**
    *   `크롤링(20개 수집) -> AI 병렬 분석 -> 점수 정렬 -> 상위 3개 반환`으로 이어지는 고품질 추천 워크플로우 완성.
*   **백엔드-프론트엔드 동기화:**
    *   프론트엔드(`api.js`)와 백엔드(`main.py`) 간의 통신 함수(`fetchProfileFromState` 등)를 최종 복구하여 서비스 전체 흐름 정상화.

## 8. 서버 통합 및 통신 오류 해결 (2026-04-09 추가)
*   **프론트엔드-백엔드 단일 서버 통합 (`app/main.py`):**
    *   포트 분리(8000/8080)로 인한 API 연결 실패를 방지하기 위해 백엔드(`FastAPI`)가 직접 `frontend` 폴더를 정적 파일로 서빙하도록 수정.
    *   이미지(assets)의 404 로드 에러를 막기 위해 `FRONTEND_DIR` 경로 지정 시 `.resolve()`를 사용하여 절대 경로로 확정.
*   **스타일 명칭 규격 동기화 (`app/main.py`):**
    *   백엔드 응답의 `primary_style` 값이 영어 코드(`modern_minimal`)로 반환되어 프론트엔드의 한글 데이터 객체(`PERSONAS`) 매칭에 실패하는 문제 해결.
    *   `fashion_config.py`의 `STYLE_LABELS` 객체를 참조하여 프론트엔드 응답 직전에 한글 라벨(`모던/미니멀`)로 자동 치환하도록 변경.
*   **프론트엔드 세션 캐싱 방어 (`js/utils.js`):**
    *   이전 포트(8080) 정보가 브라우저 세션 스토리지에 남아 API 통신이 실패하는 현상을 방어.
    *   `getApiBaseUrl()` 호출 시 캐시된 주소의 포트와 현재 접속 중인 주소(`window.location.port`)가 일치하지 않으면 캐시를 무시하고 최신 주소를 반환하도록 예외 처리 추가.
